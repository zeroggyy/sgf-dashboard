# WORKLOG

## 2026-07-16｜主題二 UI 交付追蹤與工作記錄功能

### 本次版本重點

- 版本方向：主題二「SGF 介面進度控制台」持續完善，從流程追蹤延伸至 UI 項目筆記與工作記錄。
- 資料來源：主題二 Google Sheet 分頁 `SGF_UI_DataBase`。
- 主題二 API：沿用既有 Google Apps Script Web App URL 與 API Key。
- Google Sheet 新增欄位：Q 欄，欄名為 `備註`。

### 已完成的功能

- 批次篩選改為依 Google Sheet 的「優先」欄位動態產生。
  - 支援數字格式，例如 `1`、`2`、`3`。
  - 支援文字格式，例如 `P1`～`P100`。
  - 第一批、第二批、第三批維持原本中文顯示；新增批次會自動顯示為「第 N 批」。
  - 批次數量、篩選結果與詳細視窗內容會同步更新。
- 主題二頁首時間格式與主題一統一。
  - 顯示年份、當週週次、今日日期與星期。
  - 主題二僅保留藍色週次強調色。
- 製作流程視覺化。
  - 各階段加入線性圖示：企劃、程式、驗證、拆圖、換皮、驗收、完成。
  - 階段卡片文字與數量層級放大，改善閱讀性。
  - 點擊階段後顯示外側括號選取標記。
  - 再次點擊目前選中的階段可取消篩選。
- 主題切換介面統一使用黑／米白／灰階。
  - 選中：黑底米白字。
  - 未選中：米白底黑字。
  - hover：淡灰底黑字。
  - 移除主題一紅色與主題二藍色對切換抽屜的影響。
- 詳細視窗改為筆記卡片方向。
  - 項目名稱作為主要標題。
  - 機制分類、批次、目前階段改為 tag。
  - 日期整理為簡潔資訊列。
  - 相關檔案路徑保留在底部。
- 新增工作記錄功能。
  - 詳細視窗新增可編輯的「工作記錄」文字框。
  - 「儲存記錄」按鈕移至工作記錄標題列右側。
  - 移除多餘的同步說明與底部提示文字。
  - 儲存時送出 `updateNote` POST 操作。
  - 依 `rowNumber` 寫回對應 Google Sheet 資料列。
- 主題二 GAS 新增寫回 API。
  - 檔案：`google-apps-script-theme2.js`。
  - 新增 `doPost(e)`。
  - 新增 `updateNote` action。
  - 寫入欄位依標題尋找 `備註`、`工作記錄` 或 `記錄`，目前對應 Q 欄。
  - 寫回操作仍需要 API Key 驗證。

### 需要注意的項目

- 必須將最新版 `google-apps-script-theme2.js` 貼回主題二 Google Sheet 的 Apps Script 編輯器。
- Apps Script 更新後必須重新部署網頁應用程式，否則前端仍只能讀取，無法儲存工作記錄。
- Google Sheet Q 欄標題必須是 `備註`，或使用 API 支援的替代標題 `工作記錄`／`記錄`。
- 寫回資料依 API 回傳的 `rowNumber` 判斷，不能任意刪除或重新排序資料列後再使用舊頁面；如有大幅調整資料列，建議重新整理頁面。
- 目前前端與 GAS 已完成語法檢查，但工作記錄的實際寫回仍需要在 GAS 重新部署後進行一次瀏覽器實測。
- API Key 仍會出現在前端請求流程中；目前屬於小型專案的基本防護，不應視為真正的機密保存方案。
