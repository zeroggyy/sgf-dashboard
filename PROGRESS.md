# 主題三進度筆記｜武器音效與角色語音追蹤

更新日期：2026-07-24

## 已完成

- 主題三名稱：`SGF 武器音效進度控制台`。
- 已串接 Google Sheet 與 Apps Script Web App。
- 首頁採摘要載入；點開單一武器後才讀取動作與角色語音明細，避免未來資料量增加時一次下載全部資料。
- 武器卡分別顯示音效、語音的「已確認率」與階段分布。
- 語音明細已改成「動作 × 角色」資料列，可用角色下拉選單與狀態篩選交叉查看。
- 語音編輯可個別更新角色的語音狀態與目前語音。
- 討論紀錄仍使用同一張 `音效討論紀錄` 分頁，但已區分音效／語音與對應角色；音效只看音效與共用紀錄，語音只看該角色與共用紀錄。
- 已加入「同步角色語音項目」按鈕：依武器動作與啟用角色，只新增缺少的組合，不覆寫既有資料。

## 正式狀態流程

```text
未開始 → 待製作 → 已製作 → 已確認
                      ↑
待修改 ────────────────┘

未開始 → 不需製作
```

可用狀態：

1. 未開始：初始狀態，尚未判斷或處理。
2. 待製作：已確定需要製作，等待完成。
3. 已製作：製作完成，等待檢查。
4. 待修改：檢查後打回重修。
5. 已確認：檢查通過。
6. 不需製作：已判定不需要製作。

`無內容` 只屬於網頁提示，不寫入 Google Sheet。

## 資料分頁

- `確認總表`：第 3 列起，A 欄武器名稱、D 欄音效風格。
- 同名武器分頁：第 3 列起，A 動作編號、D 指令、G 音效狀態、H 音效調整需求。
- `角色清單`：角色 ID、角色名稱、啟用；啟用可填 `TRUE`、`啟用`、`是`、`YES` 或 `1`。
- `角色語音進度`：由同步功能建立，記錄武器 × 動作 × 角色的語音狀態、語音編號、備註與更新時間。
- `音效討論紀錄`：武器名稱、動作編號、日期、留言人、討論內容、討論類型、角色 ID、角色名稱。

## 下一次接續前必做

目前 Google Apps Script 的已部署版本尚未包含最後的狀態轉換修正；因此各武器分頁 G 欄的下拉選單仍是舊的四種狀態。

1. 用本機的 `google-apps-script-theme3.js` 完整取代 Apps Script 現有程式碼。
2. 儲存後：部署 → 管理部署作業 → 編輯目前 Web App → 建立新版本 → 部署。
3. 回到網頁並重新整理。
4. 按一次「同步角色語音項目」。

同步後會更新每一把武器分頁的 G 欄，以及 `角色語音進度` 的語音狀態欄：

- `待確認` → `未開始`
- `最終確認` → `已確認`
- 下拉選單更新為：未開始、待製作、已製作、待修改、已確認、不需製作。

即使尚未填寫角色清單，同步仍會先執行上述狀態與下拉選單更新；待填好啟用角色後再次同步，才會新增角色語音資料列。
